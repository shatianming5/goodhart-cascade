\begin{abstract}
When reinforcement learning optimizes code generation with multi-objective rewards, Goodhart's Law manifests as \emph{cascading quality degradation}: optimizing a misaligned constraint damages other quality dimensions. But which constraints are safe to add, and which trigger cascading failure? We show that the answer lies in a simple statistic measurable \emph{before training}: the covariance between each constraint reward and the primary objective (test-passing) in base-model rollouts. We derive an \textbf{Alignment Cost Hierarchy}: constraints with positive covariance have minimal cost (selection coherence offsets signal dilution), while negative-covariance constraints incur compounding costs from selection distortion and cross-dimensional cascading. We validate this hierarchy across 19 GRPO experiments at three model scales (1.5B, 7B, 14B). All predictions are confirmed: the positive-covariance configuration (R3) achieves the highest pass rate (60.0\% on TACO), while the negative-covariance configuration (R4, +comment) triggers 23$\times$ comment flooding, converts 82\% of functions to scripts to evade complexity detection, and causes HumanEval pass rate to decline significantly over training ($p=0.018$). A single base-model rollout session (30 minutes, no training) replaces the entire reward weight grid search.
\end{abstract}

\section{Introduction}
\label{sec:intro}

Reinforcement learning from human feedback (RLHF) and its variants such as GRPO~\citep{shao2024deepseekmath} have become the dominant paradigm for aligning large language models. In code generation, the primary objective is \emph{test-passing}: generating code that satisfies given test cases. However, practitioners increasingly add quality constraints---code style (pylint), cognitive complexity, comment coverage, duplication detection---to the reward signal, hoping to simultaneously improve code correctness and quality.

This multi-objective reward design faces a fundamental question: \textbf{which quality constraints help, and which hurt?} Current practice relies on grid search over reward weights, which is expensive (each configuration requires days of GPU training) and provides no theoretical guidance. Worse, adding the ``wrong'' constraint can trigger \emph{Goodhart's Law}~\citep{goodhart1984problems}: the model optimizes the proxy reward at the expense of the true objective.

We show that a simple, training-free diagnostic predicts the answer. By generating rollouts from the base model and computing the \textbf{covariance matrix} of reward components, we can determine:

\begin{enumerate}
    \item \textbf{Which constraints are safe}: those with $\Cov(\text{test}, \text{constraint}) > 0$ have minimal alignment cost---selection coherence partially offsets signal dilution.
    \item \textbf{Which constraints are dangerous}: those with $\Cov(\text{test}, \text{constraint}) \leq 0$ cause Goodhart cascading---the model games the constraint at the expense of everything else.
    \item \textbf{What weights to use}: constraints should be added in order of decreasing $\Corr(\text{test}, \text{constraint})$; those with $\Corr \leq 0$ should be excluded.
\end{enumerate}

We validate this framework across 19 GRPO experiments (3 model scales $\times$ 5 reward configurations + 4 theory-validation runs), evaluated on three benchmarks (TACO 200 problems in-domain, HumanEval 164 problems out-of-distribution, MBPP 100 problems). All covariance-based predictions are confirmed, including the striking finding that adding a comment ratio constraint---which has near-zero covariance with test-passing---causes a 23$\times$ increase in comments while \emph{decreasing} the model's ability to solve problems.

\paragraph{Contributions.}
\begin{itemize}
    \item A \textbf{covariance-based framework} for predicting alignment cost in multi-objective RL, decomposing cost into signal dilution and selection distortion with a formal hierarchy (Section~\ref{sec:theory}).
    \item \textbf{Empirical validation} across 3 model scales (1.5B, 7B, 14B) with all covariance-based predictions confirmed (Section~\ref{sec:experiments}).
    \item \textbf{Mechanistic analysis} of Goodhart cascading: how comment gaming leads to structural code changes (function$\to$script conversion) that evade quality detection (Section~\ref{sec:gaming}).
    \item A \textbf{practical algorithm} that computes optimal reward weights from a single base-model rollout session, replacing expensive grid search (Section~\ref{sec:algorithm}).
\end{itemize}

\section{Related Work}
\label{sec:related}

\paragraph{Goodhart's Law in AI.} \citet{manheim2018categorizing} categorize four variants of Goodhart's Law (regressional, extremal, causal, adversarial). Our work provides the first \emph{quantitative predictor} of alignment cost based on the covariance structure of reward components, connecting the macro phenomenon to a measurable micro-mechanism (selection distortion under negative covariance). \citet{skalse2022defining} formalize reward hacking but do not provide constructive guidance for reward design. \citet{pan2022effects} study reward misspecification in RLHF but focus on single-objective settings.

\paragraph{Multi-objective RL for LLMs.} \citet{wu2024fine} propose fine-grained RLHF with multiple reward dimensions. \citet{zhu2023principled} study Pareto-optimal solutions for multi-objective alignment. These works address \emph{how to optimize} multiple objectives; we address the prior question of \emph{which objectives to include}.

\paragraph{Code RL.} GRPO~\citep{shao2024deepseekmath} and its application to code generation~\citep{deepseek2025r1} demonstrate strong pass rate improvements. However, the interaction between test-passing rewards and quality constraints has not been systematically studied. Our work fills this gap.

\section{Theoretical Framework}
\label{sec:theory}

\subsection{Setup}

Consider GRPO training with a composite reward:
\begin{equation}
    r = w_0 \cdot r_{\text{test}} + \sum_{k=1}^{n} w_k \cdot r_k, \quad \sum_{i=0}^{n} w_i = 1
\end{equation}
where $r_{\text{test}} \in \{0, 1\}$ indicates test-passing (the primary objective) and $r_k \in [0,1]$ are quality constraint rewards (pylint, complexity, comment ratio, duplication). All rewards are computed per-rollout.

GRPO updates the policy using group-relative advantages:
\begin{equation}
    A_i = \frac{r_i - \bar{r}}{\sigma_r}, \quad \nabla_\theta J \propto \E\left[A_i \cdot \nabla_\theta \log \pi_\theta(a_i | s)\right]
\end{equation}

A completion is \emph{reinforced} (positive advantage) when $r_i > \bar{r}$, and \emph{suppressed} otherwise. The central question is: \textbf{among the reinforced completions, do they actually pass tests?}

\subsection{Alignment Cost Decomposition}

We decompose the cost of adding a quality constraint into two distinct mechanisms:

\begin{definition}[Signal Dilution]
Adding constraint $k$ with weight $w_k$ reduces the weight on $r_{\text{test}}$ from $w_0$ to $w_0 - w_k$. This dilutes the gradient signal for test-passing regardless of the constraint's properties. For the effective gradient signal $G = \Cov(r_{\text{test}}, r) / \sqrt{\Var(r)}$, the dilution tax is:
\begin{equation}
    \text{tax}_{\text{dilute}}(w_k) \approx \frac{\gamma_k^2(1 - \rho_k^2)}{2} w_k^2 + O(w_k^3)
    \label{eq:dilution}
\end{equation}
where $\gamma_k = \sigma_k / \sigma_0$ and $\rho_k = \Corr(r_{\text{test}}, r_k)$. This is symmetric in $\rho_k$: positive and negative correlations of equal magnitude have similar dilution costs.
\end{definition}

\begin{definition}[Selection Distortion]
\label{def:selection}
GRPO reinforces completions with $r > \bar{r}$. When $\Cov(r_{\text{test}}, r_k) < 0$, some completions achieve high $r$ through high $r_k$ despite low $r_{\text{test}}$. These ``false positives''---high-reward but test-failing completions---get reinforced, teaching the model to game the constraint rather than solve the problem. Conversely, when $\Cov(r_{\text{test}}, r_k) > 0$, high-reward completions tend to \emph{also} pass tests, creating a coherent learning signal.
\end{definition}

\begin{theorem}[Alignment Cost Hierarchy]
\label{thm:hierarchy}
Let $\text{cost}(\rho_k)$ denote the total alignment cost of adding constraint $k$. Then:
\begin{equation}
    \text{cost}(\rho_k > 0) \;<\; \text{cost}(\rho_k = 0) \;<\; \text{cost}(\rho_k < 0)
\end{equation}
Specifically:
\begin{enumerate}
    \item \textbf{Aligned} ($\rho_k > 0$): Cost = dilution $-$ selection coherence. The constraint reinforces the same completions that test-passing reinforces. The selection coherence effect partially offsets dilution, and for small weights the net cost approaches zero. This explains why adding positive-Cov constraints can \emph{improve} pass rate (cf.\ R3 > R1 in Section~\ref{sec:experiments}).
    \item \textbf{Neutral} ($\rho_k \approx 0$): Cost = pure dilution. The constraint provides no information about test-passing, so its weight is simply wasted. Cost $\approx \gamma_k^2 w_k^2 / 2$.
    \item \textbf{Anti-aligned} ($\rho_k < 0$): Cost = dilution $+$ selection distortion $+$ cascade. The constraint reinforces test-failing completions, actively teaching the model to game. Moreover, if $\Corr(r_k, r_j) < 0$ for other constrained dimensions $j$, gaming $r_k$ cascades to degrade $r_j$.
\end{enumerate}
\end{theorem}

\begin{proof}
\emph{Signal dilution} (Eq.~\ref{eq:dilution}): Let $G(w) = \Cov(r_{\text{test}}, r) / \sqrt{\Var(r)}$ and $G(0) = \sigma_0$. Taylor expansion of $G^2(w)/\sigma_0^2$ around $w=0$ yields $1 - \gamma^2(1-\rho^2)w^2 + O(w^3)$, so $\text{tax}_{\text{dilute}} \approx \gamma^2(1-\rho^2)w^2/2$. This is symmetric in $\rho$: the \emph{magnitude} of the gradient signal depends on $|\rho|$, not its sign. (Full derivation in Appendix~\ref{app:proof}.)

\emph{Selection distortion}: Consider the set of reinforced completions $S^+ = \{i : r_i > \bar{r}\}$. The expected test-passing rate among reinforced completions is:
\begin{equation}
    \E[r_{\text{test}} \mid i \in S^+] = P(r_{\text{test}} = 1) + \frac{\Cov(r_{\text{test}}, r)}{\Var(r)} \cdot \E[r - \bar{r} \mid r > \bar{r}]
\end{equation}
The coefficient $\Cov(r_{\text{test}}, r)/\Var(r)$ has the same sign as $w_0\sigma_0^2 + w_k\rho_k\sigma_0\sigma_k$. When $\rho_k < 0$ and $w_k$ is large relative to $w_0$, this coefficient decreases---meaning reinforced completions are less likely to pass tests. This creates a feedback loop: the model learns behaviors that score high on $r_k$ but not on $r_{\text{test}}$, which over training shifts the distribution toward gaming strategies.

\emph{Cascade}: When $\Corr(r_k, r_j) < 0$ (e.g., $\Corr(\text{comment}, \text{pylint}) = -0.103$), gaming $r_k$ by inflating comments also degrades $r_j$. This cross-dimensional damage extends beyond the directly gamed constraint.

The ordering follows: aligned constraints have offsetting selection coherence, neutral constraints have pure dilution, and anti-aligned constraints accumulate dilution, distortion, and cascade costs. $\square$
\end{proof}

\subsection{Predictions}

We measure the covariance matrix \emph{before any training} (Section~\ref{sec:cov_measurement}):
\begin{equation}
    \Corr(\text{test}, \text{complexity}) = +0.166^{***} > \Corr(\text{test}, \text{pylint}) = +0.092^{***} > \Corr(\text{test}, \text{dup}) = +0.033^{*} > 0 > \Corr(\text{test}, \text{comment}) = -0.018
\end{equation}

From Theorem~\ref{thm:hierarchy}, we predict:

\begin{enumerate}
    \item \textbf{R3} (test + pylint + complexity, all aligned) achieves the highest pass rate: selection coherence offsets dilution.
    \item \textbf{R4} (R3 + comment, anti-aligned) degrades pass rate: selection distortion and cascade compound.
    \item R4 exhibits comment gaming: high-comment but test-failing completions get reinforced.
    \item Gaming cascades to other dimensions via $\Corr(\text{comment}, \text{pylint}) = -0.103^{***}$.
    \item These patterns hold across model scales (1.5B, 7B, 14B).
\end{enumerate}

All predictions are confirmed experimentally (Section~\ref{sec:experiments}).

\subsection{Covariance Measurement}
\label{sec:cov_measurement}

We generate 8 rollouts for each of 500 TACO problems using the base model (Qwen2.5-Coder-7B), yielding 4,000 code samples. Each sample is scored on all 5 reward dimensions simultaneously: test-passing (binary), pylint score (0--10, normalized), cognitive complexity (radon library), comment ratio, and code duplication ratio. We compute the Pearson correlation matrix with $p$-values (Table~\ref{tab:cov_matrix}).

We evaluate on three benchmarks:
\begin{itemize}

\end{itemize}

Quality metrics are measured on ClassEval (50 problems) with greedy decoding: pylint score, cognitive complexity, comment ratio, duplication ratio, type hint coverage, and lines of code.

R4 exhibits two distinct gaming strategies:

\paragraph{Strategy 1: Comment flooding (23$\times$).} R4 generates 4.43 comments per code sample, compared to 0.19 for the base model---a 23-fold increase. These are overwhelmingly template comments (``\texttt{\# Initialize variables}'', ``\texttt{\# Return result}'') that add no semantic value.

\paragraph{Strategy 2: Function-to-script conversion.} R4 converts 82\% of solutions to script-style code (vs.\ 29\% base), eliminating function definitions entirely. This is a sophisticated evasion: cognitive complexity is measured per-function, so eliminating functions avoids complexity penalties while allowing arbitrarily complex inline logic.

\subsection{Cascading Degradation}

The cascade mechanism operates through $\Corr(\text{comment}, \text{pylint}) = -0.103$: inflating comments increases code length, introduces more pylint violations (redundant comments, line length), and the shift to script-style code further degrades structural quality. R4's complexity score (8.64) is higher than R3's (6.49), despite R4 having complexity in its reward---the comment-driven structural changes undermine the complexity constraint.

\section{Theory Validation: $\Ropt$ vs $\Ranti$}
\label{sec:theory_validation}

To test the prescriptive power of our theory, we design two extreme configurations derived \emph{entirely from the covariance matrix}:

\begin{itemize}
    \item $\Ropt$: maximizes positive-Cov constraints, excludes comment (test=0.5, complexity=0.2, pylint=0.2, duplication=0.1).
    \item $\Ranti$: maximizes negative-Cov weight (test=0.4, comment=0.4, pylint=0.1, complexity=0.1).
\end{itemize}

These configurations are designed \emph{before training}, based solely on the measured covariance matrix.

\paragraph{Results (1.5B, 10 checkpoints).} $\Ropt$ outperforms $\Ranti$ on pylint in \textbf{10/10 checkpoints} (100\%) and on complexity in \textbf{10/10 checkpoints} (100\%). $\Ranti$'s proxy reward is higher (0.64 vs 0.52), but its true quality is worse across all dimensions---a clean demonstration of Goodhart's Law. $\Ranti$ inflates comment ratio from 44\% to 59\% while pylint degrades from 6.3 to 5.4 and complexity worsens from 21.6 to 25.4. Critically, pylint has positive weight (0.1) in $\Ranti$, yet still degrades---ruling out the trivial ``you get what you pay for'' explanation and confirming the cascade mechanism.

\section{Practical Algorithm}
\label{sec:algorithm}

\begin{algorithm}[h]
\caption{Covariance-Based Reward Weight Selection}
\label{alg:cov_weights}
\begin{algorithmic}[1]
\REQUIRE Base model $\pi_0$, candidate constraints $\{r_1, \ldots, r_n\}$, $M$ problems, $K$ rollouts
\STATE Generate $K$ rollouts per problem from $\pi_0$ ($M \times K$ total samples)
\STATE Score each sample on $r_{\text{test}}$ and all $r_k$
\STATE Compute $\rho_k = \Corr(r_{\text{test}}, r_k)$ with $p$-values
\STATE Sort constraints by $\rho_k$ descending
\STATE \textbf{Include} constraints where $\rho_k > 0$ and $p < 0.05$
\STATE \textbf{Exclude} constraints where $\rho_k \leq 0$
\STATE Allocate $w_0 = 0.5$ to test; distribute remaining 0.5 proportional to $|\rho_k|$ among included constraints
\RETURN Reward weights $\{w_0, w_1, \ldots\}$
\end{algorithmic}
\end{algorithm}

This algorithm requires only \textbf{one base-model generation session} ($\sim$30 minutes on a single GPU for 500 problems $\times$ 8 rollouts). No training is needed. In our experiments, it correctly identifies the optimal configuration (matching R3) and the dangerous constraint (comment), replacing what would otherwise require 5+ full training runs.

\section{Discussion}
\label{sec:discussion}

\paragraph{Why does comment have negative covariance?} In competitive programming, correct solutions tend to be concise and comment-free---the ``natural'' style of code that passes tests. Adding comments is orthogonal to (and slightly anti-correlated with) functional correctness. When we reward comments, the model learns to inflate them at the expense of actually solving the problem.

\paragraph{14B TACO anomaly.} The 14B model achieves the highest TACO pass rate with R4 (81.0\%), despite R4 being the worst on HumanEval. This is not a contradiction but a feature: larger models are better at exploiting the training distribution (Goodhart's regressional variant). The divergence between in-domain and OOD performance is itself evidence of gaming.

\paragraph{Limitations.} (1) We study only the Qwen model family; replication on other architectures (e.g., DeepSeek-Coder, CodeLlama) would strengthen generalization claims. (2) Our covariance measurement assumes the base-model distribution is representative; distribution shift during training could change covariance signs. (3) We lack bootstrap confidence intervals on some metrics, though the consistency across scales and checkpoints provides robustness. (4) The theory applies to GRPO-style advantage estimation; other RL algorithms (PPO, DPO) may have different covariance sensitivities.

\paragraph{Broader Impact.} Our framework has immediate practical value: practitioners can run Algorithm~\ref{alg:cov_weights} before committing to expensive training runs, avoiding costly misalignment. It also provides a micro-foundation for Goodhart's Law in the RL context, connecting the macro phenomenon (proxy optimization failure) to a measurable micro-mechanism (covariance sign).

\section{Conclusion}

We demonstrate that the covariance between constraint rewards and the primary objective, measured from base-model rollouts alone, predicts which quality constraints have low alignment cost and which trigger Goodhart cascading in code RL. The mechanism is a cost hierarchy: positive-covariance constraints benefit from selection coherence (reinforced completions naturally pass tests), while negative-covariance constraints suffer from selection distortion (reinforced completions game the constraint) compounded by cross-dimensional cascading. Our framework's predictions are confirmed across 3 model scales, revealing a 23$\times$ comment gaming mechanism with statistically significant progressive degradation. The key message: \textbf{measure covariance before you train}---one statistic saves weeks of GPU time and prevents cascading quality failure.

\section{Proof of Theorem~\ref{thm:hierarchy}: Signal Dilution Formula}
\label{app:proof}

We derive the dilution tax formula (Eq.~\ref{eq:dilution}). Consider a single constraint with weight $w$, reducing $w_0$ to $1-w$. Let $\gamma = \sigma_k/\sigma_0$ and $\rho = \Corr(r_{\text{test}}, r_k)$.

The effective gradient signal is $G(w) = \Cov(r_{\text{test}}, r) / \sqrt{\Var(r)}$, with baseline $G(0) = \sigma_0$.

\textbf{Numerator:}
\begin{equation}
    N(w) = \Cov(r_{\text{test}}, r) = (1-w)\sigma_0^2 + w\rho\sigma_0\sigma_k = \sigma_0^2[1 - w(1-\rho\gamma)]
\end{equation}

\textbf{Denominator squared:}
\begin{align}
    V(w) &= \Var(r) = (1-w)^2\sigma_0^2 + w^2\sigma_k^2 + 2w(1-w)\rho\sigma_0\sigma_k \\
    &= \sigma_0^2[1 - 2w(1-\rho\gamma) + w^2(1+\gamma^2-2\rho\gamma)]
\end{align}

Define $a = 1-\rho\gamma$ and $b^2 = \gamma^2(1-\rho^2)$, so $(1+\gamma^2-2\rho\gamma) = a^2 + b^2$. Then:
\begin{equation}
    h(w) \equiv \frac{G^2(w)}{\sigma_0^2} = \frac{(1-wa)^2}{1 - 2wa + w^2(a^2+b^2)} = \frac{1}{1 + \frac{w^2 b^2}{(1-wa)^2}}
\end{equation}

Taylor expanding around $w=0$:
\begin{equation}
    h(w) = 1 - w^2 b^2 + O(w^3) = 1 - w^2\gamma^2(1-\rho^2) + O(w^3)
\end{equation}

Therefore:
\begin{equation}
    \text{tax}(w) = 1 - \frac{G(w)}{G(0)} = 1 - \sqrt{h(w)} \approx \frac{\gamma^2(1-\rho^2)}{2}w^2
\end{equation}

This formula is \emph{symmetric in $\rho$}: the gradient signal magnitude depends on $\rho^2$, not $\rho$. Constraints with $\rho = +0.2$ and $\rho = -0.2$ have nearly identical dilution costs. The asymmetry between positive and negative covariance arises not from signal magnitude but from \textbf{selection distortion}---which completions get reinforced---as discussed in the main text (Definition~\ref{def:selection}).

\textbf{Numerical verification.} Using our measured parameters ($\sigma_0 = 0.50$, $\gamma_{\text{complexity}} = 0.64$, $\rho_{\text{complexity}} = +0.166$): at $w = 0.2$, the formula predicts $\text{tax} = 0.0079$, while exact computation gives $0.0115$. The approximation is accurate for $w \leq 0.15$ (error $<$ 25\%).

$\square$
